## Dashboard Philosophy

Dashboards make organizational state visible. When the right information is visible to the right people at the right time, the organization can:

- Make better decisions based on current information
- Identify problems early before they escalate
- Understand how organizational effort translates to outcomes
- Track progress toward strategic objectives
- Celebrate achievements

Sandy creates and maintains dashboards that provide visibility into organizational operations.

## Dashboard Principles

**1. One Source of Truth**
There is one dashboard for each domain (projects, performance, risks). People look there first.

**2. Current Information**
Dashboards show current state, typically updated daily or within 24 hours.

**3. Appropriate Detail**
High-level overview for executive view; detail available for those who need it.

**4. Actionable Information**
Information on dashboard tells someone what action to take, if any.

**5. Clear Ownership**
Each dashboard has an owner responsible for accuracy and currency.

**6. Easy to Use**
Dashboards are designed for easy understanding; don't require extensive interpretation.

**7. Mobile Friendly**
Dashboards can be viewed on devices used by stakeholders.

## Core Organizational Dashboards

### 1. Project Portfolio Dashboard

**Purpose**: Overview of all active projects and their status

**Audience**: Managing Director, project owners, Sandy

**Key Information**:
- List of all active projects
- Current phase of each project
- Schedule status (on time, at risk, delayed) with % complete
- Quality status
- Resource status
- Critical milestones
- Recent completions

**View Options**:
- List view: All projects in table format
- Timeline view: Projects on timeline showing phases and milestones
- Status summary: Project status overview (# on time, at risk, delayed)
- Resource view: Resource allocation across projects

**Update Frequency**: Daily

**Drill-down**: Click on project to see detailed project status

### 2. Organizational Performance Dashboard

**Purpose**: Track organizational performance against strategic objectives

**Audience**: Managing Director, team leads, Sandy

**Key Information**:
- Strategic objectives and progress toward each
- Key performance indicators and trends
- Quality metrics (% of work meeting standards, rework rates)
- Schedule performance (% projects on time, schedule variance)
- Resource utilization
- Capacity vs. demand

**View Options**:
- Executive summary: High-level strategic performance
- Detailed view: All metrics with trends
- Trend view: Performance trends over time
- Comparison view: Performance vs. goals

**Update Frequency**: Weekly with daily updates to real-time metrics

**Drill-down**: Click on metric to see underlying data and details

### 3. Risk Dashboard

**Purpose**: Make organizational risks visible and track mitigation

**Audience**: Managing Director, project owners, Sandy

**Key Information**:
- Identified risks
- Risk status (new, active, mitigated, resolved)
- Risk assessment (probability, impact, priority)
- Mitigation status
- Escalated risks
- Resolved risks

**View Options**:
- Active risks: Currently active risks that need attention
- High-impact risks: Risks with highest potential impact
- At-risk projects: Projects where risks are emerging
- Risk trend: How risk profile is changing

**Update Frequency**: As risks change or at least weekly

**Drill-down**: Click on risk to see details, mitigation, and status

### 4. Team Capacity Dashboard

**Purpose**: Track team capacity and allocation across work

**Audience**: Sandy, project owners

**Key Information**:
- Available team capacity (hours, by specialist)
- Allocated capacity (hours by project/priority)
- Unallocated capacity
- Capacity conflicts (overallocation)
- Capacity trends over time
- Bottlenecks (overly-utilized specialists)

**View Options**:
- Current capacity: Today's or this week's allocation
- Projected capacity: Planned allocation for coming weeks
- Specialist view: Individual specialist capacity
- Priority view: Capacity allocated to each priority

**Update Frequency**: Daily

**Drill-down**: Click on allocation to see specific projects and tasks

### 5. Knowledge Repository Dashboard

**Purpose**: Track knowledge management and organizational learning

**Audience**: Sandy, team members

**Key Information**:
- Knowledge repository size (# of documents)
- Recent additions (newly captured knowledge)
- Knowledge being used (most accessed documents)
- Knowledge gaps (questions asked vs. documented answers)
- Knowledge updates (recently updated documents)

**View Options**:
- Browse by category: Find knowledge by category
- Search: Find knowledge by keyword
- Recent additions: Latest captured knowledge
- Most used: Most frequently accessed knowledge
- My knowledge: Knowledge relevant to current role

**Update Frequency**: As knowledge is added or accessed

**Drill-down**: Click on document to view full knowledge

### 6. Automation Dashboard

**Purpose**: Track automation opportunities and implementation status

**Audience**: Sandy, automation developers

**Key Information**:
- Automation opportunities identified
- Automation implementations in progress
- Completed automations
- Automation effectiveness (time saved, quality improvement)
- Automation failures or issues

**View Options**:
- Opportunity pipeline: Automations being considered or planned
- Implementation status: Automations in development
- Production automations: Active automations
- Effectiveness: Impact and ROI of automations
- Issues: Automations with problems

**Update Frequency**: Weekly

**Drill-down**: Click on automation to see details and effectiveness

## Dashboard Design Principles

### Information Hierarchy

Most important information is most prominent:

1. **Red zones**: Critical issues requiring immediate action (alerts, at-risk projects, critical risks)
2. **Yellow zones**: Items that need attention (approaching deadlines, quality concerns)
3. **Green zones**: Status quo (on-track projects, normal operations)
4. **Gray zones**: Background information (completed projects, resolved risks)

### Visual Design

Dashboards use clear visual language:

- **Color coding**: Red/yellow/green for status; clear and consistent
- **Icons**: Simple symbols for quick recognition
- **Numbers**: Big, clear numbers for key metrics
- **Trends**: Charts showing performance over time
- **Comparisons**: Showing actual vs. goal or expected

### Clarity Over Elegance

Dashboards prioritize clarity:

- Use simple, clear language
- Explain acronyms and jargon
- Define metrics clearly
- Show data source and update time
- Make drill-down obvious

### Appropriate Frequency

Different dashboards have different update needs:

- **Real-time**: Dashboards updated constantly (capacity, real-time metrics)
- **Daily**: Most dashboards updated daily or within 24 hours
- **Weekly**: Strategic dashboards may update weekly
- **Monthly**: Long-term trend dashboards updated monthly

### Drill-Down Capability

High-level dashboards support drill-down to detail:

- Click a summary statistic to see underlying data
- Click a project status to see project details
- Click a risk to see mitigation details
- Click a metric to see historical trend

## Creating New Dashboards

When new dashboards are needed:

### Step 1: Define Purpose

- What question does the dashboard answer?
- Who needs to use it?
- What decisions will be made based on it?

### Step 2: Identify Metrics

- What metrics answer the core question?
- What supporting information helps interpretation?
- What is the appropriate level of detail?

### Step 3: Determine Data Source

- Where does the data come from?
- Is the data automatically collected or manually entered?
- Is it current and accurate?
- Who maintains the data?

### Step 4: Design Layout

- What information is most important and should be most prominent?
- How is information organized and grouped?
- What color coding and visual language is used?
- How does it drill down to detail?

### Step 5: Build and Test

- Create dashboard prototype
- Test with intended users
- Refine based on feedback
- Launch when ready

### Step 6: Maintain

- Establish update frequency
- Designate ownership
- Monitor for accuracy
- Refine based on usage

## Dashboard Governance

### Dashboard Ownership

Each dashboard has:

- **Owner**: Person responsible for accuracy, currency, and usefulness
- **Data Sources**: Where data comes from
- **Update Frequency**: How often dashboard is updated
- **Access**: Who can view and edit
- **Support**: Who can answer questions about dashboard

### Dashboard Standards

All dashboards follow consistent standards:

- Standard color coding for status
- Consistent terminology and definitions
- Clear data source and update time labeled
- Clear drill-down paths
- Professional appearance

### Data Quality

Dashboard data must be:

- **Accurate**: Data reflects reality
- **Complete**: All relevant data is included
- **Current**: Data updated at defined frequency
- **Consistent**: Data is consistently defined and measured
- **Auditable**: Data sources can be traced

### Dashboard Evolution

Dashboards evolve over time:

- New metrics are added as needs change
- Outdated metrics are retired
- Layout is improved based on usage
- Frequency may be adjusted
- New dashboards are created as needs emerge

## Anti-Patterns to Avoid

### Dashboard Overload

Too much information makes dashboards useless. Keep focused on:

- High-level executive dashboard: 5-7 key metrics
- Operational dashboard: 10-15 key metrics
- Detailed dashboard: More detail available via drill-down

### Metric Obsession

Not everything that can be measured should be on a dashboard. Focus on:

- Metrics that drive decisions
- Metrics that track strategic objectives
- Metrics that identify problems
- Metrics that show organizational health

### Lack of Action

Dashboards are only valuable if they drive action. Ensure:

- Dashboards highlight items needing action
- Clear paths exist to take action
- Dashboard data is acted upon
- Follow-up occurs on flagged items

### Stale Data

Dashboards lose credibility when data is outdated. Ensure:

- Data is updated at promised frequency
- Outdated data is marked as such
- Systems are in place to catch update failures
- Ownership is clear

### Wrong Audience

Different people need different dashboards. Ensure:

- Executive dashboard summarizes for leaders
- Operational dashboard shows detailed status for operators
- Team dashboard shows relevant information for team
- Individual dashboard shows work needing action
