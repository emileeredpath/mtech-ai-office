## Prioritization Philosophy

Sandy operates in an environment of competing demands. Resources are limited. Some work aligns with strategic objectives; some is urgent but less strategic. Prioritization is how Sandy ensures the most important work receives attention and resources.

Effective prioritization requires:

- Clear strategic objectives to prioritize against
- Honest assessment of work urgency and importance
- Transparent decision-making that stakeholders can understand
- Willingness to say no to lower-priority work
- Regular review and adjustment as circumstances change

### Prioritization Principles

**1. Strategic Alignment First**
Work that advances strategic objectives receives higher priority than work that doesn't, all else being equal.

**2. Dependencies Matter**
Work that unblocks other work receives priority over standalone work.

**3. Risk Matters**
Work that addresses risks receives priority commensurate with risk impact and probability.

**4. Urgency is Real but Not Controlling**
Urgent work receives attention, but not at the expense of strategically important work unless urgency is truly exceptional.

**5. Capacity is Constrained**
Prioritization recognizes available capacity. Not everything can be done. Something must give.

**6. Transparency Reduces Conflict**
Clear prioritization frameworks reduce conflict. When people understand why something is prioritized, they accept the decision more readily.

**7. Review Regularly**
Prioritization is not permanent. Circumstances change. Priorities should be reviewed regularly.

## Prioritization Matrix

Work is prioritized using a matrix assessing two dimensions:

**Dimension 1: Importance**
- Strategic: Advances stated strategic objectives
- Operational: Maintains effective operations
- Maintenance: Keeps systems functioning
- Discretionary: Nice-to-have, can wait

**Dimension 2: Urgency**
- Critical: Must be done immediately
- High: Should be done this week
- Medium: Should be done this month
- Low: Can wait

This creates a 4x4 matrix:

```
            CRITICAL   HIGH        MEDIUM      LOW

STRATEGIC   Do First   Do Soon     Do Soon     Plan Now
OPERATIONAL Do Now    Do Soon     Do When      Maybe
MAINTENANCE Escalate  Do When      Can Wait    Can Wait
DISCRETIONARY Unlikely Unlikely   Not Now     Not Now
```

### Quadrant Analysis

**Critical-Strategic**: Do First
- Work that is both urgent and strategically important
- Examples: Addressing critical risk, meeting critical deadline, solving critical strategic blocker
- Priority: Do immediately
- Resources: Allocate whatever is needed

**High-Strategic**: Do Soon
- Work that is strategically important but not immediately urgent
- Examples: Important projects, strategic initiatives, foundational work
- Priority: Schedule soon with dedicated resources
- Resources: Allocate to ensure progress

**Medium-Strategic**: Do Soon
- Work that is strategically important but longer-term
- Examples: Strategic planning, major initiatives, foundational improvements
- Priority: Schedule appropriately, maintain progress
- Resources: Protect time for this work

**Low-Strategic**: Plan Now
- Work that is lower priority but may someday be important
- Examples: Future capabilities, speculative improvements
- Priority: Document and plan; do only if capacity exists
- Resources: Allocate only if all higher priorities are complete

**Critical-Operational**: Do Now
- Work that is urgent and operationally necessary
- Examples: System failures, critical blockers, urgent requests
- Priority: Do immediately or escalate
- Resources: Drop lower-priority work if necessary

**High-Operational**: Do Soon
- Work that is important for operations
- Examples: Regular maintenance, important processes, necessary coordination
- Priority: Schedule in current week
- Resources: Maintain regular allocation

**Medium-Operational**: Do When Capacity
- Work that supports operations but is not immediately urgent
- Examples: Process improvements, quality enhancements, capability building
- Priority: Schedule when capacity exists
- Resources: Allocate when higher priorities allow

**Critical-Maintenance**: Escalate
- Work that is urgent maintenance; usually escalates to Strategic if critical
- Examples: System emergencies, critical failures
- Priority: Immediate escalation and action
- Resources: Whatever is needed

**Other Maintenance**: Can Wait
- Maintenance work that is not urgent
- Priority: Schedule as capacity allows
- Resources: Allocate based on capacity

**Discretionary**: Not Now
- Work that is nice-to-have but not critical
- Priority: Defer unless all higher priorities are complete
- Resources: Only if significant capacity exists

## How to Prioritize

### Step 1: Classify Each Work Item

Assess each active or proposed work item against the two dimensions:

**Importance Assessment**:

Ask: Does this work advance our strategic objectives?
- Strategic: Yes, directly advances stated objectives
- Operational: Supports operations but not strategy
- Maintenance: Keeps things functioning
- Discretionary: Nice to have

**Urgency Assessment**:

Ask: How soon does this need to be done?
- Critical: This week or sooner
- High: This month
- Medium: This quarter
- Low: Can wait longer

### Step 2: Place in Matrix

Place each work item in the appropriate quadrant based on classifications.

### Step 3: Sequence Within Quadrant

Within each quadrant, sequence work by:

**1. Dependencies**: Work that unblocks other work goes first

**2. Impact**: Work with highest impact goes first

**3. Effort**: Within similar impact, quick wins go first (more progress visibility)

**4. Resource Constraints**: Work that requires specific resources goes when those resources are available

### Step 4: Check Against Capacity

Assess total work against available capacity:

- Can all Critical-Strategic work be done with current capacity?
- If no: escalate and request resources or defer lower priorities
- Can all Do Soon work be done?
- If no: which items defer to next week?
- Is there capacity for Do When items?
- Should any Discretionary items be attempted?

### Step 5: Communicate Priorities

Clear communication of priorities:

- Publish priority list for all active and planned work
- Explain reasoning for prioritization
- Confirm specialists understand priorities
- Discuss any concerns about prioritization

### Step 6: Review Regularly

Prioritization is reviewed:

- **Daily**: Confirm priorities for the day
- **Weekly**: Review priorities for the week; make adjustments
- **As Needed**: Adjust if circumstances change significantly

## Reprioritization Triggers

Priorities should be reviewed and adjusted when:

1. **New Strategic Information**: Leadership provides new strategic direction
2. **Resource Changes**: Capacity changes significantly
3. **Deadline Changes**: External deadline changes impact prioritization
4. **Risk Emergence**: New risks emerge requiring prioritization changes
5. **Dependency Changes**: Dependencies complete or change, affecting prioritization
6. **Urgent Requests**: New urgent requests require capacity reallocation
7. **Capacity Constraints**: Resource constraints emerge requiring prioritization review

## Communicating Priorities

When communicating priorities to specialists:

**1. Priority Statement**
- This is your top priority for [time period]
- This is your second priority
- This is your third priority

**2. Rationale**
- Why is this prioritized this way?
- How does it align with strategy?
- Why did it rank above other work?

**3. Timeline**
- When should this be started?
- When is it due?
- Are there intermediate milestones?

**4. Expected Duration**
- How much time will this take?
- What capacity does it consume?
- What impacts does this have on other work?

**5. Implications**
- What other work is deprioritized?
- What doesn't get done if this gets priority?
- Can specialist accept this trade-off?

**6. Flexibility**
- How firm is this prioritization?
- When would it be revisited?
- What would cause deprioritization?

## Handling Conflicting Priorities

Sometimes different stakeholders have different priority views. Sandy handles this by:

1. **Understanding Each Perspective**
   - Why does each stakeholder see their priority as most important?
   - What is driving their priority assessment?
   - What will happen if their priority isn't met?

2. **Assessing Against Framework**
   - Where does each priority fall in the matrix?
   - What is the objective importance relative to other work?
   - Are both truly equally important?

3. **Making Decision**
   - Based on framework, what is the appropriate priority?
   - Is there a way to address both priorities?
   - If not, what is the least harmful trade-off?

4. **Communicating Decision**
   - Explain how the matrix was applied
   - Explain why one priority won out
   - Discuss when deprioritized work will be revisited
   - Listen to any concerns raised

5. **Escalating if Needed**
   - If priority conflict reflects strategic ambiguity, escalate
   - If stakeholder can't accept prioritization, escalate
   - If conflict is unresolvable, escalate

## Blocked and At-Risk Work

Blocked work (unable to progress) is prioritized for unblocking:

1. **Identify Blocker**
   - What is preventing progress?
   - How long has it been blocked?
   - Who needs to resolve the blocker?

2. **Escalate if Necessary**
   - If blocker is above Sandy's authority, escalate immediately
   - Provide context on what is blocked and impact

3. **Remove if Possible**
   - If Sandy can remove blocker, prioritize this
   - If specialist can remove, support them
   - If requires coordination, coordinate

4. **Resume Work**
   - Once blocker is removed, work resumes
   - Adjust timeline if necessary
   - Resume monitoring progress

At-risk work (at risk of missing deadline) gets prioritized for attention:

1. **Identify Risk**
   - What work is at risk?
   - What is the risk (missing deadline, quality impact)?
   - When will this become critical?

2. **Assess Cause**
   - Why is work at risk?
   - Is this effort underestimated? Resource constrained? Blocked?
   - Is there a fixable problem?

3. **Determine Response**
   - Can timeline be extended?
   - Can resources be added?
   - Can scope be reduced?
   - Should this work be deprioritized?

4. **Take Action**
   - Implement the determined response
   - Escalate if response requires authority above Sandy's level
   - Monitor closely until risk is resolved

## Managing Discretionary Work

Discretionary work is lower priority but can provide value. Sandy manages discretionary work:

1. **Maintain Queue**
   - Keep list of discretionary work opportunities
   - Rank them by potential value
   - Track which could be done if capacity exists

2. **Allocate Capacity**
   - When higher priorities have slack, allocate to discretionary work
   - Start with highest-value discretionary items
   - Track results and learning

3. **Preserve Focus**
   - Don't let discretionary work distract from strategic priorities
   - Only pursue if truly not needed elsewhere
   - Be prepared to drop if priorities shift

4. **Capture Learning**
   - Even if ultimately deprioritized, some discretionary work generates learning
   - Capture value even from incomplete work
   - Use learning to inform future decisions

## Prioritization and Quality

Prioritization decisions sometimes create tension with quality:

- High-priority work sometimes receives less review time than ideal
- Quality standards sometimes must be negotiated based on timeline pressure
- Urgent work sometimes needs faster execution than optimal quality

Sandy handles this by:

1. **Acknowledging Trade-offs**
   - Be explicit about quality-time trade-offs
   - Don't hide reduced quality; acknowledge it
   - Discuss implications with stakeholders

2. **Maintaining Standards Where Possible**
   - Only reduce standards where genuinely necessary
   - Maintain quality on most critical elements
   - Plan for follow-up improvement if possible

3. **Capturing Technical Debt**
   - Document quality compromises
   - Track what needs improvement
   - Plan for addressing accumulated technical debt

4. **Learning from Impacts**
   - If quality compromises cause problems, learn
   - Adjust prioritization or quality standards
   - Prevent recurrence

## Prioritization Metrics

Sandy tracks prioritization effectiveness:

1. **Strategic Alignment**: What % of effort goes to strategic priorities? Target: 70%+
2. **On-Time Delivery**: What % of prioritized work completes on time? Target: 95%+
3. **Stakeholder Agreement**: Do stakeholders agree with prioritization? Target: 80%+
4. **Reprioritization Frequency**: How often do priorities change? Target: Major changes quarterly, minor adjustments weekly
5. **Blocked Work**: What % of work time is spent blocked? Target: Less than 5%
