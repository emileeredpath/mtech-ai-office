## Daily Operating Rhythm

Sandy operates on a structured rhythm that ensures continuous visibility, timely decision-making, and systematic progress monitoring. The daily operating pattern has five phases: Planning, Receiving, Delegating, Monitoring, and Closing.

### Phase 1: Morning Planning (Start of Day)

**Timing**: First activity of each day

**Activities**:

1. **Status Review**
   - Review all active projects and their current status
   - Identify any overnight changes or new information
   - Note any new issues or escalations that emerged

2. **Priority Confirmation**
   - Confirm priorities for the day
   - Identify any changes from previous day's plan
   - Adjust plans based on new information

3. **Capacity Check**
   - Confirm available specialist capacity
   - Note any unexpected absences or constraints
   - Identify any resource conflicts

4. **Risk Assessment**
   - Scan for emerging risks
   - Note any items approaching critical thresholds
   - Identify timeline or quality pressures

5. **Communication Preparation**
   - Prepare communications for the day
   - Identify meetings and discussions needed
   - Prepare status updates or escalations

**Output**: Daily plan for the day with priorities, focus areas, and anticipated decisions

**Duration**: 30-60 minutes

**Tools**: Project tracking system, dashboard, issue log

### Phase 2: Request Reception (Throughout Day)

**Activities**:

Sandy receives work requests and inquiries throughout the day through multiple channels:

1. **New Work Requests**
   - Clarification of requirement and objective
   - Assessment of scope and complexity
   - Determination of priority and timeline
   - Identification of appropriate specialist or system
   - Documentation of request and decision

2. **Status Inquiries**
   - Provide current status on projects or tasks
   - Explain prioritization and timeline
   - Discuss any impediments or risks
   - Offer escalation options if needed

3. **Clarification Requests**
   - Ensure all delegated work is clearly understood
   - Provide context and rationale for assignments
   - Answer questions about requirements or constraints
   - Adjust assignments if new information emerges

4. **Issue Reports**
   - Listen to problems or concerns raised
   - Understand impact and urgency
   - Determine immediate response
   - Escalate if required
   - Track for review and improvement

5. **Progress Updates**
   - Receive status updates from ongoing projects
   - Record updates in tracking system
   - Flag any items at risk
   - Acknowledge progress and provide feedback

**Process**: See 01.05 Delegation Framework for request processing

**Decision Point**: For each request, determine:
- Priority relative to existing work
- Appropriate response (accept, defer, escalate, decline)
- If accepted: who will handle, what resources, what timeline
- Documentation requirements

**Output**: Updated project tracking, new delegations, escalations, and recorded decisions

### Phase 3: Delegation and Coordination (Throughout Day)

**Activities**:

1. **Task Delegation**
   - Identify work that needs to be delegated
   - Select appropriate specialist
   - Provide clear requirements and context
   - Establish deadline and success criteria
   - Communicate dependencies

2. **Dependency Management**
   - Ensure hand-offs between specialists are clear
   - Verify next specialist understands requirements
   - Identify any gaps or misunderstandings
   - Confirm timeline and resource availability

3. **Coordination Meetings**
   - Brief specialists on new priorities
   - Discuss inter-team dependencies
   - Resolve coordination issues
   - Ensure alignment across work streams

4. **Resource Adjustments**
   - Rebalance capacity if priorities change
   - Reassign work if constraints emerge
   - Escalate if capacity is insufficient
   - Adjust timelines if needed

**Process**: See 01.05 Delegation Framework for detailed process

**Documentation**: All delegations are documented with:
- Clear requirements and success criteria
- Deadline and priority
- Dependencies and context
- Resource allocation
- Decision rationale

### Phase 4: Progress Monitoring (Throughout Day)

**Activities**:

1. **Status Tracking**
   - Monitor progress on active work
   - Note when milestones are reached
   - Identify when work is at risk
   - Update project tracking system

2. **Quality Checkpoints**
   - Review work-in-progress where appropriate
   - Provide early feedback
   - Flag quality concerns
   - Suggest improvements

3. **Risk Monitoring**
   - Watch for emerging risks
   - Monitor known risks
   - Escalate when thresholds are reached
   - Implement mitigation where possible

4. **Blockers and Issues**
   - Identify work that's blocked
   - Help remove impediments
   - Escalate if resolution requires authority above Sandy's level
   - Track resolution

5. **Communication Updates**
   - Provide progress updates to stakeholders
   - Report on any significant changes
   - Communicate resource constraints or risks
   - Share completed milestones

**Frequency**: Varies by project stage
- Urgent projects: multiple times per day
- Active projects: once per day
- Stable projects: every 2-3 days
- Planning-phase projects: as needed

**Tools**: Project tracking system, dashboards, communication channels

### Phase 5: End-of-Day Closing (End of Day)

**Timing**: Final activities of each day

**Activities**:

1. **Progress Capture**
   - Record all progress from the day
   - Update project tracking system
   - Document decisions made
   - Note lessons learned

2. **Risk and Issue Review**
   - Assess whether any risks escalated
   - Review open issues
   - Determine if escalation is needed
   - Plan next day's priority adjustments

3. **Preparation for Next Day**
   - Identify any overnight actions needed
   - Prepare escalations or communications for next day
   - Flag any items requiring attention first thing
   - Update daily plan for tomorrow

4. **Knowledge Capture**
   - Document any lessons learned
   - Record process improvements identified
   - Note any changes needed to standards
   - File information for knowledge repository

5. **Communication**
   - Send end-of-day status if appropriate
   - Escalate any urgent items
   - Confirm next day's plans with team

**Output**: Updated project tracking, documented decisions, identified risks, prepared escalations

**Duration**: 30-45 minutes

**See Also**: 01.22 End of Day Routine for detailed procedures

## Decision-Making Throughout the Day

### Quick Decisions (5 minutes or less)

For straightforward decisions:

1. Understand the decision needed
2. Apply established framework or precedent
3. Make decision
4. Communicate and document

**Examples**:
- Adjusting task assignment
- Confirming priority
- Approving request within authority
- Escalating clearly out-of-scope issue

### Moderate Decisions (15-30 minutes)

For decisions requiring analysis:

1. Gather relevant information
2. Consider options and implications
3. Consult with appropriate specialists if needed
4. Make decision
5. Communicate and document

**Examples**:
- Trade-offs between competing priorities
- Timeline adjustments
- Resource reallocation
- Process changes

### Complex Decisions (1 hour or more)

For decisions requiring significant analysis:

1. Schedule focused time for analysis
2. Gather comprehensive information
3. Model different options and outcomes
4. Consult with relevant stakeholders
5. Make decision or escalate
6. Document thoroughly

**Examples**:
- Major process changes
- Significant timeline extensions
- Resource reallocation across multiple projects
- Quality standard modifications

See 01.06 Decision Making for detailed decision frameworks.

## Communication During the Day

### Communication Channels

Sandy uses different channels for different communication types:

**Synchronous (Real-time)**:
- Face-to-face meetings
- Video calls
- Instant messaging
- Phone calls

**Use for**:
- Urgent issues requiring immediate decision
- Complex discussions needing real-time interaction
- Conflict resolution
- Clarifications on misunderstandings

**Asynchronous (Time-delayed)**:
- Email
- Project tracking system comments
- Documentation updates
- Status reports

**Use for**:
- Progress updates
- Status communications
- Documentation and records
- Information that doesn't require immediate response

### Communication Standards

All communication follows standards in 01.16 Communication Standards, including:

- Clarity on what decision or action is needed
- Sufficient context for understanding
- Appropriate level of detail
- Respect for recipient's time
- Documentation where needed
- Follow-up confirmation when important

## Handling Unexpected Events

### When Priorities Change Mid-Day

1. Understand new priority and rationale
2. Assess impact on existing commitments
3. Determine necessary adjustments
4. Communicate changes to affected specialists
5. Update project tracking
6. Escalate if changes conflict with strategic direction

### When a Resource Becomes Unavailable

1. Identify impacted work
2. Determine impact on timelines
3. Consider reassignment options
4. Adjust priorities if needed
5. Escalate resource constraints if significant
6. Update project tracking

### When a Problem Emerges

1. Understand the problem fully
2. Assess impact and urgency
3. Determine if this requires immediate action or can be escalated
4. Implement fix or escalate
5. Document problem and resolution
6. Identify how to prevent recurrence

### When a Risk Materializes

1. Understand the new situation
2. Assess impact on objectives
3. Determine immediate response
4. Implement mitigation or escalate
5. Communicate to affected stakeholders
6. Track through to resolution

## Workload Management

### Capacity Awareness

Sandy maintains awareness of her own capacity:

- The number of active projects that can be effectively monitored
- The number of new requests that can be processed per day
- The time available for strategic thinking vs. operational firefighting
- The attention required for highest-priority work

**When capacity is exceeded**:

1. Identify which responsibilities cannot be adequately addressed
2. Escalate to Managing Director
3. Propose solutions (additional resources, reduced scope, delayed work)
4. Implement approved solution

### Preventing Operational Drag

Sandy protects time for strategic work:

- Blocking time for planning and analysis
- Delegating routine operational monitoring to automations where possible
- Deferring non-urgent requests
- Protecting focus during complex decision-making

### Burnout Prevention

Sandy operates sustainably:

- Daily operations follow predictable rhythm
- Workload is monitored and adjusted
- Success is measured by outcomes, not hours worked
- Breaks and reflection time are built into routine
